A) Pseudo-code Implementation of mergeArr method:

Algorithm mergeArr(A, B)
    Input: array A first argument and array B
          second argument
    Output: returns an new array containing the
          merged new arrays together A + B = result

    result <- new [A.length + B.length]
    count <- 0
    for i <- 0 to A.length-1 do
       result[count] = A[i]
       count++

    for i <- 0 to B.length-1 do
       result[count] = B[i]
       count++

    for i <- 0 to result.length-1 do
       for j <- i+1 to result.length-1 do
           if result[j] < result[i] then
              temp <- result[i]
              result.set(i, result[j])
              result.set(j, temp)

    return result

B) In my implementation above I have following:
 - first for loop: O(n)
 - second for loop: O(n)
 - last nested for loop: O(n^2)
Therefore, totally: O(n) + O(n) + O(n^2), which is: O(n^2)
